## 2. Flutter 分层详解

### 2.1 Clean Architecture 三层

```
presentation/  ← 用户能看到的一切：页面、BLoC、Widget
domain/        ← 纯 Dart 接口，不依赖任何框架
data/          ← 接口实现，处理网络请求和数据转换
core/          ← 跨层共享的基础设施
```

**依赖方向**: `presentation → domain ← data`（domain 不依赖任何层）

### 2.2 BLoC 状态管理

每个功能模块的 BLoC 由三个文件组成：

```
blocs/<feature>/
├── <feature>_bloc.dart    # 业务逻辑，事件→状态转换
├── <feature>_event.dart   # 输入事件定义
└── <feature>_state.dart   # 输出状态定义
```

**当前 5 个 BLoC 模块：**

| BLoC | 事件 | 状态 | 职责 |
|------|------|------|------|
| `AuthBloc` | `AuthCheckRequested`, `AuthLoginRequested`, `AuthLogoutRequested` | `AuthInitial`, `AuthLoading`, `AuthAuthenticated`, `AuthUnauthenticated`, `AuthError` | 登录/注册/Token管理 |
| `ChatBloc` | `ChatMessageSent`, `ChatStreamStarted`, `ChatStreamChunkReceived`, `ChatStreamDone` | `ChatInitial`, `ChatLoading`, `ChatLoaded`, `ChatStreaming`, `ChatError` | AI文字对话+SSE流式 |
| `ChatHistoryBloc` | `ChatHistoryLoaded`, `ChatHistoryRefreshed` | `ChatHistoryInitial`, `ChatHistoryLoading`, `ChatHistoryLoaded`, `ChatHistoryError` | 对话列表+详情 |
| `HealthBloc` | `HealthRecordCreated`, `HealthRecordsLoaded`, `HealthRecordDeleted` | `HealthInitial`, `HealthLoading`, `HealthLoaded`, `HealthError` | 健康记录CRUD |
| `VoiceBloc` | `VoiceConnect`, `VoiceDisconnect`, `VoiceSendAudioChunk`, `VoiceSendImageChunk`, `VoiceInterrupt`, `VoiceReceiveMessage` | `VoiceInitial`→`VoiceConnecting`→`VoiceConnected`→`VoiceListening`→`VoiceProcessingInput`→`VoiceReceivingText/Audio`→`VoiceDone` | 语音/视频通话全生命周期 |

### 2.3 依赖注入 (get_it)

`injection.dart` 管理所有依赖：

```
LocalStorage (单例)
    ↓
ApiClient, WebSocketClient (懒加载)
    ↓
AuthRepository, HealthRepository, ChatRepository (懒加载)
    ↓
AuthBloc, ChatBloc, ChatHistoryBloc, HealthBloc, VoiceBloc (工厂)
```

### 2.4 路由设计 (go_router)

```
/ (启动页, 带动画)
├── /login (登录)
├── /call (语音/视频通话, 无底部导航)
├── /chat-history/:id (对话详情, 无底部导航)
└── ShellRoute (底部导航栏)
    ├── /chat (AI 咨询)
    ├── /health-records (健康记录)
    ├── /chat-history (对话历史)
    └── /profile (个人中心)
```

`ShellRoute` 包裹的 4 个 Tab 共享同一个 `BottomNavigationBar`。

### 2.5 平台适配模式

Flutter 通过 Dart 条件导入实现同一接口的多平台实现：

```dart
// 声明时
import 'audio_recorder_stub.dart'
    if (dart.library.html) 'audio_recorder_web.dart'
    if (dart.library.io) 'audio_recorder_android.dart';
```

| 模块 | 基类/接口 | Web 实现 | Android 实现 | Stub |
|------|-----------|----------|-------------|------|
| AudioRecorder | `AudioRecorderBase` | AudioContext + ScriptProcessor | MethodChannel → AudioRecord | 空实现 |
| AudioPlayer | `AudioPlayerBase` | AudioContext + BufferSource | MethodChannel → AudioTrack | 空实现 |
| CameraCapture | `CameraCaptureBase` | getUserMedia + Canvas | MethodChannel → CameraX | 空实现 |

## 3. 后端分层详解

### 3.1 路由层 (routers/)

| 路由文件 | 前缀 | 端点 |
|----------|------|------|
| `auth.py` | `/api/auth` | register, login, me |
| `health.py` | `/api/health` | records CRUD |
| `consult.py` | `/api/consult` | chat, chat/stream, conversations |
| `voice.py` | `/api/consult/voice` | chat, chat/stream, ws (WebSocket) |

### 3.2 服务层 (services/)

| 服务 | 职责 |
|------|------|
| `auth_service.py` | 密码哈希、Token 生成/验证 |
| `health_service.py` | 健康记录 CRUD |
| `ai_service.py` | 调用 DashScope API（文本对话、语音对话、实时语音） |

### 3.3 数据模型 (models/)

```
User ──1:N──→ HealthRecord
User ──1:N──→ Conversation ──1:N──→ Message
```

| 模型 | 表 | 关键字段 |
|------|-----|----------|
| `User` | users | id(UUID), phone, hashed_password |
| `HealthRecord` | health_records | id, user_id, record_type, value, status |
| `Conversation` | conversations | id, user_id, title |
| `Message` | messages | id, conversation_id, role(user/assistant), content |

### 3.4 测试架构

使用 **SQLite 内存隔离**，测试间数据自动清空：

```python
# conftest.py 关键逻辑
os.environ["DATABASE_URL"] = "sqlite:///./test_isolated.db"  # 覆盖数据库
# ...
@pytest.fixture(autouse=True)
def clean_database():
    """每个测试前 DELETE 所有表"""
```

测试文件：`test_auth.py`, `test_health.py`, `test_ai.py`

## 4. 通信协议

### 4.1 HTTP REST API

```
POST /api/auth/login     → {"phone": "...", "password": "..."}
                         ← {"access_token": "eyJ...", "token_type": "bearer"}

POST /api/consult/chat   → {"messages": [{"role":"user","content":"..."}], "conversation_id": null}
                         ← {"reply": "...", "conversation_id": "uuid"}

POST /api/consult/chat/stream  → 同上 (Accept: text/event-stream)
                               ← SSE: data: {"content":"..."}\n\n ... data: [DONE]\n\n
```

### 4.2 SSE 流式输出

Flutter 端 `sse_client_web.dart` 使用浏览器原生 `EventSource`。非 Web 平台使用 `sse_client_stub.dart` 存根。

后端 `StreamingResponse` 格式：
```
data: {"choices":[{"delta":{"content":"你"}}]}\n\n
data: {"choices":[{"delta":{"content":"好"}}]}\n\n
...
data: [DONE]\n\n
```

### 4.3 WebSocket 实时通话

```
Flutter ──WS──→ Backend ──WS──→ DashScope Realtime
         token认证          API Key认证
```

详细协议见 [语音通话实现](./语音通话实现.md)。
