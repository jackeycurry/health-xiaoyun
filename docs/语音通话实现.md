# 语音/视频通话实现

## 概述

实时语音通话通过 WebSocket 桥接 Flutter 客户端和阿里云 DashScope Qwen-Omni Realtime API。支持语音双向对话、视频帧输入、打断、转录持久化。

---

## 1. 架构图

```
┌──────────────┐     WebSocket       ┌──────────────┐     WebSocket      ┌─────────────────┐
│  Flutter App │ ◄──────────────────► │  FastAPI      │ ◄───────────────► │  DashScope       │
│              │   JSON Messages      │  voice.py     │   DashScope API   │  Realtime API    │

│  Camera      │                      │               │  response.cancel  │                  │
└──────────────┘                      └──────────────┘                   └─────────────────┘
```

## 2. 连接建立流程

### 2.1 Flutter 发起连接

```dart
// call_page.dart
_voiceBloc?.add(VoiceConnect(token));

// voice_bloc.dart → WebSocket 连接
_webSocketClient.connect(token);
// URL: ws://localhost:8002/api/consult/voice/ws?token=xxx
```

### 2.2 后端握手 (voice.py)

```
1. WebSocket 接受连接 + JWT 认证
2. 创建 Conversation 记录 (title="语音通话")
3. 连接 DashScope Realtime API
4. 接收 session.created
5. 发送 session.update (配置 VAD/音色/指令)
6. 接收 session.updated
7. 发送 "connected" 消息给 Flutter (含 conversation_id)
8. 启动双向桥接任务
```

### 2.3 DashScope session.update 配置

```python
{
    "type": "session.update",
    "session": {
        "modalities": ["text", "audio"],
        "voice": "Tina",                    # 音色
        "input_audio_format": "pcm",        # 输入 16kHz mono 16bit PCM
        "output_audio_format": "pcm",       # 输出 24kHz mono 16bit PCM
        "instructions": "你是健康小云...",   # 系统提示词
        "turn_detection": {
            "type": "semantic_vad",         # 语义 VAD
            "threshold": 0.5,               # 检测阈值
            "silence_duration_ms": 1000     # 静音判定时长
        }
    }
}
```

## 3. 消息协议

### 3.1 Flutter → Backend

| type | 数据 | 说明 |
|------|------|------|
| `audio` | `{"type":"audio","data":"<base64 pcm>"}` | 实时音频块 (16kHz mono PCM) |
| `image` | `{"type":"image","data":"<base64 jpeg>"}` | 视频帧 (JPEG, 建议 480p) |
| `commit` | `{"type":"commit"}` | 手动提交音频缓冲区 |
| `interrupt` | `{"type":"interrupt"}` | 用户点击打断 AI |
| `ping` | `{"type":"ping"}` | 15s 心跳保活 |
| `stop` | `{"type":"stop"}` | 挂断通话 |

### 3.2 Backend → Flutter

| type | 数据 | 说明 |
|------|------|------|
| `connected` | `{"type":"connected","conversation_id":"uuid"}` | 连接就绪 |
| `text` | `{"type":"text","data":"你好"}` | AI 文本流 (增量) |
| `audio` | `{"type":"audio","data":"<base64 pcm>"}` | AI 音频 delta |
| `speech_started` | `{"type":"speech_started"}` | VAD检测到用户开始说话 |
| `speech_stopped` | `{"type":"speech_stopped"}` | VAD检测到用户停止说话 |
| `user_text` | `{"type":"user_text","data":"..."}` | 用户语音转录结果 |
| `ai_text` | `{"type":"ai_text","data":"..."}` | AI 完整回复文本 |
| `done` | `{"type":"done"}` | AI 本轮回复结束 |
| `error` | `{"type":"error","data":"..."}` | 错误信息 |

### 3.3 Backend → DashScope

| type | 说明 |
|------|------|
| `session.update` | 配置会话参数 |
| `input_audio_buffer.append` | 追加音频到缓冲区 (需 `event_id`) |
| `input_image_buffer.append` | 追加图像到缓冲区 (需 `event_id`) |
| `input_audio_buffer.commit` | 提交缓冲区 |
| `input_audio_buffer.clear` | 清除缓冲区 (打断后防回声) |
| `response.cancel` | 取消正在进行的 AI 回复 |

### 3.4 DashScope → Backend (关键事件)

| type | 说明 |
|------|------|
| `session.created` / `session.updated` | 会话配置确认 |
| `response.created` | AI开始生成回复 |
| `response.audio.delta` | AI 音频流 |
| `response.audio_transcript.delta` | AI 文本转录流 |
| `response.audio_transcript.done` | AI 完整转录 (→ 持久化) |
| `response.done` | AI 回复完成 |
| `input_audio_buffer.speech_started` | 检测到用户语音 |
| `input_audio_buffer.speech_stopped` | 用户语音结束 |
| `conversation.item.input_audio_transcription.completed` | 用户转录 (→ 持久化) |

## 4. 回声处理策略

### 问题

模拟器上 `AcousticEchoCanceler.isAvailable()` 返回 false，扬声器输出直接被麦克风采集，导致 AI 回声循环。

### 解决方案：三级音频控制 + 缓冲区清除

```
┌──────────────────────────────────────────────┐
│  状态              麦克风控制        目的      │
├──────────────────────────────────────────────┤
│  AI 处理/回复中    mute()           完全静音  │
│  等待用户说话      gateOn()         噪声门    │
│  用户说话中        gateOff()        全量收音  │
└──────────────────────────────────────────────┘
```

**后端配合**：每次 `response.cancel` 后立即 `input_audio_buffer.clear`，防止残留回声被 VAD 自动提交。

### 真机 vs 模拟器

| 环境 | AEC | 音频路由 | 回声风险 |
|------|-----|----------|----------|
| 真机 | 硬件 AEC 可用 | 听筒 (小声) | 低 |
| 模拟器 | 不可用 | 扬声器内录 | 高 |
| 浏览器 | 浏览器 AEC | 扬声器 | 低 |

## 5. 打断机制

### 5.1 语音打断 (VAD 自动)

```
用户说话 → DashScope VAD 检测 → speech_started
→ voice.py: response.cancel + input_audio_buffer.clear
→ Flutter: VoiceListening → gateOff → 重新收音
```

### 5.2 手动打断 (点击按钮)

```
用户点击 → _interruptAi() → VoiceInterrupt 事件
→ voice_bloc: 发送 {"type":"interrupt"}
→ voice.py: _cancel_and_clear() → cancel + clear buffer
→ Flutter: VoiceListening → gateOff → 用户说话
```

## 6. 音频录制/播放实现

### 6.1 Android

```
Flutter (Dart) ──MethodChannel──→ MainActivity.kt
                                    ├── AudioRecord (16kHz mono PCM)
                                    │   └── AcousticEchoCanceler (硬件AEC)
                                    └── AudioTrack (24kHz mono PCM)
                                        └── USAGE_VOICE_COMMUNICATION (听筒)
```

### 6.2 Web

```
Flutter (Dart) ──JS eval──→ 浏览器 API
                              ├── getUserMedia({echoCancellation:true})
                              ├── AudioContext (16000Hz)
                              ├── ScriptProcessor (PCM→Base64)
                              └── AudioContext (24000Hz)
                                  └── BufferSource→destination
```

## 7. 视频帧采集

### 流程

```
CameraCapture.startCapture()
    ↓
Android: camera 插件 → CameraController → ImageStream → JPEG
Web:     getUserMedia({video:true}) → Canvas → toDataURL('image/jpeg')
    ↓
VoiceSendImageChunk(base64Jpeg)
    ↓
WebSocket: {"type":"image","data":"<base64>"}
    ↓
voice.py → input_image_buffer.append
    ↓
DashScope 视觉模型处理
```

**限制**：JPEG ≤500KB (Base64前)，建议 480p ~ 720p，1帧/秒。图像必须在音频之后发送。

## 8. 对话持久化

语音通话自动保存转录文本：

```
DashScope 事件                         保存内容
──────────────────────────────────────────────────
response.audio_transcript.done    →   AI 回复文本 → Message(role="assistant")
input_audio_transcription.        →   用户语音文本 → Message(role="user")
  completed
```

通话结束后，Flutter 跳转到 `/chat?conversationId=xxx`，用户可查看完整文字记录。
