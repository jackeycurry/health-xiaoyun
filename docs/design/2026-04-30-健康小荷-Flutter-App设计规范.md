# 健康小云 App 设计规范

## 1. 设计理念

**品牌定位**: 温暖亲切的 AI 健康管家
**核心理念**: 像朋友一样关心你的健康，用专业知识和温暖语气陪伴用户
**视觉调性**: 专业但不失温度，医疗但不冰冷

---

## 2. 色彩系统

### 主色调
| 用途 | 色值 | 说明 |
|------|------|------|
| 主色-蓝绿 | `#4ECDC4` | 清新健康感，品牌主色 |
| 主色-深蓝 | `#2D9CDB` | 专业可信，辅助强调 |
| 背景渐变起点 | `#E8F8F7` | 浅薄荷绿，温暖开场 |
| 背景渐变终点 | `#FFFFFF` | 纯白底，干净舒适 |

### 功能色
| 用途 | 色值 | 说明 |
|------|------|------|
| 成功/正向 | `#52C41A` | 健康指标良好 |
| 警告/注意 | `#FFB800` | 需要关注 |
| 危险/严重 | `#FF4D4F` | 需及时就医 |
| AI 消息背景 | `#F0FAFB` | 浅蓝绿气泡 |
| 用户消息背景 | `#4ECDC4` | 品牌色气泡 |
| 用户消息文字 | `#FFFFFF` | 白色文字 |

### 文字色
| 用途 | 色值 | 说明 |
|------|------|------|
| 主标题 | `#1A1A1A` | 品牌名、页面标题 |
| 正文 | `#333333` | 主要内容 |
| 次要文字 | `#666666` | 时间戳、辅助说明 |
| 占位符 | `#999999` | 输入框提示语 |

## 5. 页面结构

### 5.1 整体架构
```
┌─────────────────────────────┐
│      顶部品牌区 (72px)       │
│  [小云Icon] 健康小云  [菜单]  │
├─────────────────────────────┤
│                             │
│                             │
│      消息列表区域            │
│      (可滚动)               │
│                             │
│                             │
├─────────────────────────────┤
│   [记录] [   输入框   ] [发送]│
│        底部输入区 (60px)      │
└─────────────────────────────┘
```

### 5.2 顶部品牌区 (72px)
- 左侧：小云 icon (40x40px) + "健康小云" 文字
- 右侧：菜单按钮 (头像/设置入口)
- 背景：白色 + 底部 1px 分割线 `#E0E0E0`

### 5.3 消息列表区
- 系统欢迎消息置顶
- 消息气泡间距：12px
- 消息与顶部间距：16px
- 消息与底部输入框间距：16px

### 5.4 底部输入区 (60px 基础高度，可扩展)
- 左侧：健康记录快捷入口 (40x40px 圆形按钮)
- 中间：文本输入框 (圆角 24px，背景 `#F5F5F5`)
- 右下角：发送按钮 (仅输入框有文字时显示)

## 6. 组件设计

### 6.1 消息气泡

**AI 助手消息 (左对齐)**
```
┌─────────────────────────────┐
│ 🤖                          │
│ ┌─────────────────────────┐ │
│ │ 消息内容文字             │ │
│ │ 第二行文字               │ │
│ └─────────────────────────┘ │
│                    10:30    │
└─────────────────────────────┘
```
- 头像：32x32px，圆形，显示 🤖 或小云形象
- 气泡：圆角 16px，左圆角 4px，背景 `#F0FAFB`，边框 1px `#E0F4F3`
- 最大宽度：屏幕宽度 - 80px
- 文字：`#333333`，16sp

**用户消息 (右对齐)**
```
┌─────────────────────────────┐
│                    10:31    │
│ ┌─────────────────────────┐ │
│ │ 消息内容文字             │ │
│ └─────────────────────────┘ │
└─────────────────────────────┘
```
- 气泡：圆角 16px，右圆角 4px，背景 `#4ECDC4`
- 最大宽度：屏幕宽度 - 80px
- 文字：`#FFFFFF`，16sp

### 6.2 语音输入动画
- 按钮：长按显示录音中状态
- 波形：3条动态波形，绿色 `#4ECDC4`
- 时长：显示已录制时间

### 6.3 健康记录卡片 (快捷入口)
- 形状：圆形，40x40px
- 背景：白色 + 阴影
- 图标：心率/血压/血糖等健康图标
- 位置：输入框左侧

### 6.4 输入框
- 高度：48px (圆角 24px)
- 背景：`#F5F5F5`
- 提示文字："请输入您的健康问题..."
- 发送按钮：圆形，背景 `#4ECDC4`，白色箭头图标

### 6.5 底部弹窗/菜单
用于：健康记录、历史记录、个人中心

## 7. 页面流程

### 7.1 启动流程
```
启动 → 品牌欢迎页(可选) → 登录页(未登录) → 聊天首页
```

### 7.2 聊天首页功能入口
- **健康记录入口** → 健康记录列表页
- **历史记录入口** → 聊天历史列表页
- **个人中心入口** → 个人中心页

### 7.3 页面列表
1. **登录页** - 手机号 + 验证码
2. **聊天首页** - AI 咨询主界面
3. **健康记录页** - 记录列表 + 新增记录
4. **历史记录页** - 聊天历史列表
5. **个人中心页** - 头像、设置、关于

## 8. AI 人设定义

**称呼**: "你"（亲切感）
**语气**: 温暖、专业、易懂
**回复风格示例**:
- 你好！我是健康小云，你的健康管家~ 有什么健康问题可以问我哦！
- 您的血压有点偏高呢，建议平时少盐多运动，如有不适要及时就医哦。
- 这个症状持续多久了？了解更多信息我可以给你更好的建议~

## 9. 技术实现建议

### Flutter 核心依赖
- `flutter_bloc` - 状态管理
- `dio` - 网络请求
- `shared_preferences` - 本地存储
- `flutter_tts` - 语音播报
- `speech_to_text` - 语音识别
- `image_picker` - 图片选择

### API 对接
- 基础 URL: `http://localhost:8000` (开发环境)
- 认证: Bearer Token (JWT)
- 登录: `POST /api/auth/login`
- AI 聊天: `POST /api/consult/chat/stream`
- 健康记录: `GET/POST /api/health/records`
