# 部署与打包

## 1. 环境配置

### 1.1 后端 `.env` 配置

```env
# backend/.env

# 数据库 (PostgreSQL 生产 / SQLite 测试)
DATABASE_URL=postgresql://user:password@localhost:5432/health_app

# JWT 签名密钥 (生产环境务必修改为随机字符串)
SECRET_KEY=your-production-secret-key

# 阿里云百炼 API
DASHSCOPE_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
AI_MODEL=qwen3-omni-flash

# 可选
DEBUG=False
```

### 1.2 Flutter API 地址配置

`health_xiaohe/lib/core/network/api_endpoints.dart`:

```dart
static String get baseUrl {
    if (kIsWeb) return 'http://localhost:8002';
    // 修改为你的后端地址
    return 'http://YOUR_SERVER_IP:8002';
}
```

### 1.3 Android 网络权限

`AndroidManifest.xml` 已配置：
- `INTERNET` 权限
- `RECORD_AUDIO` 权限
- `CAMERA` 权限
- `android:usesCleartextTraffic="true"` (开发环境 HTTP)

## 2. 构建打包

### 2.1 Flutter Web

```bash
cd health_xiaohe

# 构建
flutter build web

# 产物在 build/web/ 目录
# 部署到任意静态文件服务器 (Nginx / OSS / Vercel)
```

Nginx 配置示例：
```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/build/web;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### 2.2 Flutter Android APK

```bash
cd health_xiaohe

# Debug APK (开发测试)
flutter build apk --debug

# Release APK
flutter build apk --release

# AAB (Google Play)
flutter build appbundle --release
```

产物路径：
- APK: `build/app/outputs/flutter-apk/app-release.apk`
- AAB: `build/app/outputs/bundle/release/app-release.aab`

### 2.3 后端部署

**方案 A: 直接部署 (开发/小规模)**

```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8002 --workers 4
```

**方案 B: Docker**

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY backend/requirements.txt .
RUN pip install -r requirements.txt
COPY backend/ .
EXPOSE 8002
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8002", "--workers", "4"]
```

```bash
docker build -t health-xiaohe-backend .
docker run -d -p 8002:8002 --env-file backend/.env health-xiaohe-backend
```

**方案 C: Systemd 服务 (Linux)**

```ini
# /etc/systemd/system/health-xiaohe.service
[Unit]
Description=健康小荷 Backend
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/health-xiaohe/backend
EnvironmentFile=/opt/health-xiaohe/backend/.env
ExecStart=/usr/bin/uvicorn main:app --host 0.0.0.0 --port 8002 --workers 4
Restart=always

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable --now health-xiaohe
```

## 3. 数据库初始化

后端首次启动自动建表（`Base.metadata.create_all`），无需手动迁移。

```python
# database.py — 启动时自动执行
Base.metadata.create_all(bind=engine)
```

如需手动创建 PostgreSQL 数据库：

```sql
CREATE DATABASE health_app;
CREATE USER health_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE health_app TO health_user;
```

## 6. 常见问题

### 端口冲突
```bash
# 修改后端端口
uvicorn main:app --port 8003
# 同步修改 ApiEndpoints.baseUrl
```

### 模拟器无法连接后端
模拟器使用 `10.0.2.2` 访问宿主机 localhost，修改 `ApiEndpoints.baseUrl`：
```dart
return 'http://10.0.2.2:8002';  // Android 模拟器
```

### CORS 错误
后端已配置 `allow_origins=["*"]`，生产环境建议改为具体域名。

### DashScope API 401
检查 `.env` 中 `DASHSCOPE_API_KEY` 是否正确。
